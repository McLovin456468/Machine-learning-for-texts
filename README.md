# Проект: Классификация токсичных комментариев

## 1. Назначение и цель проекта

В сервисе, где пользователи могут редактировать и дополнять описания товаров (аналогично вики-сообществам), возникает необходимость модерации контента. Требуется автоматически выявлять токсичные комментарии и направлять их на дополнительную проверку.

**Цель:** построить модель классификации комментариев на позитивные и негативные (токсичные).

**Целевое требование к качеству:** значение метрики F1 не менее 0.75.

## 2. Входные данные

**Источник:** файл `/datasets/toxic_comments.csv`

**Структура данных:**

| Поле | Описание |
|------|----------|
| `text` | Текст комментария |
| `toxic` | Целевой признак (токсичность комментария) |

*Примечание:* задача бинарной классификации.

## 3. Этапы выполнения

### 3.1. Загрузка и подготовка данных
- Загрузить данные из CSV-файла.
- Проверить наличие пропусков и дубликатов. При необходимости выполнить очистку.
- Проанализировать баланс классов (распределение токсичных и нетоксичных комментариев).

### 3.2. Предобработка текстовых данных
Выполнить стандартную очистку текста:
- приведение к нижнему регистру;
- удаление пунктуации и специальных символов;
- удаление стоп-слов (опционально);
- лемматизация или стемминг (при необходимости);
- токенизация.

**Альтернативный подход:** использование BERT без явной предобработки (токенизация средствами самой модели).

### 3.3. Векторизация текста
Рассмотреть следующие методы преобразования текста в числовые признаки:
- `TfidfVectorizer` (с ограничением максимального количества признаков);
- `CountVectorizer`;
- эмбеддинги (Word2Vec, FastText);
- BERT-эмбеддинги (при использовании трансформеров).

Данные разделить на обучающую и тестовую выборки (стратифицированно с учётом целевого признака).

### 3.4. Обучение моделей и подбор гиперпараметров

Рассмотреть следующие модели:
- логистическая регрессия (базовая модель);
- случайный лес;
- градиентный бустинг (LightGBM, XGBoost, CatBoost);
- (опционально) BERT с дообучением (fine-tuning).

Для каждой модели провести подбор гиперпараметров (например, с помощью `GridSearchCV` или `RandomizedSearchCV`).

### 3.5. Оценка качества

**Основная метрика:** `F1` (среднее гармоническое между полнотой и точностью).

Дополнительные метрики:
- `precision` (точность);
- `recall` (полнота);
- `AUC-ROC`.

Выполнить проверку требования: **F1 ≥ 0.75** на тестовой выборке.

### 3.6. Анализ результатов
- Сравнить качество различных моделей.
- Проанализировать ошибки классификации (на каких типах комментариев модель ошибается).
- Сделать вывод о пригодности лучшей модели для операционной деятельности.

## 4. Выходные результаты

- Jupyter Notebook с полным кодом: загрузка, предобработка, обучение, оценка.
- Лучшая модель (сохранённая в формате `joblib` / `pickle`).
- Значение метрики F1 на тестовой выборке (не менее 0.75).
- Выводы о качестве модели и возможности её применения для автоматической модерации.

## 5. Примечания

- Возможно использование BERT (требуется выполнение локально или в среде с достаточным объёмом RAM/VRAM).
- Допускается реализация без BERT
- Основной критерий успеха: F1 ≥ 0.75.
